# Security Policy

## Supported versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a vulnerability

Please do **not** open a public GitHub issue for security vulnerabilities.

Report suspected vulnerabilities privately to the MongoDB security team at
**security@mongodb.com**, or open a private
[GitHub security advisory](https://github.com/mongodb-partners/langchain-mongodb-agent-log/security/advisories/new).

Include a description of the issue, reproduction steps, and the affected
version. You will receive an acknowledgement, and we will keep you updated as
the report is investigated and resolved.

For MongoDB's general vulnerability-reporting guidance, see
<https://www.mongodb.com/docs/manual/tutorial/create-a-vulnerability-report/>.
